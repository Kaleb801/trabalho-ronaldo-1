Perguntas

1)O que é a “Engenharia de software sala limpa”?

R= É uma abordagem que tem como foco a necessidade de agregar precisão ao software enquanto ele está em desenvolvimento


2)Qual o objetivo do teste sala limpa?

R= É validar requisitos de software demonstrando que uma amostragem estatística de caso de uso foi executada de forma bem-sucedida.


3)Qual o conceito de métodos formais?

R= Métodos formais utilizados no desenvolvimento de software, são técnicas de base matemática para descrever propriedades de sistemas. Ele proporciona uma estrutura dentro da qual as pessoas podem especificar, desenvolver e verificar sistemas de maneira sistemática, em vez de casual.


4)O que os métodos formais oferece para um sistema?

R= Oferece um modo de verificar se um sistema importante vai funcionar de acordo com a sua especificação e para que isso ocorra, o software é tratado como uma entidade matemática cuja exatidão pode ser provada com operações lógicas.


5)Em vez de testes o que a engenharia de software sala limpa utiliza?

R= É utilizado uma representação de box structure que está dividida em três tipos: Caixa-preta, Caixa de estado e Caixa-clara, que são utilizadas para encapsular o sistema em um nível de detalhe maior. Assim é realizado a análise e modelagem do projeto que tem como foco a verificação da correção, que é o principal mecanismo para localizar e remover os erros.
